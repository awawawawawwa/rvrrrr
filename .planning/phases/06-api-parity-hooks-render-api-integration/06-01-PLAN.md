**src/contexts/focus-context.ts** - FocusContext:

- `remove(id)` removes from array; if removed id was active, activeId becomes undefined
- `next()` / `previous()` rotate through the ordered array (wrapping)
- `focus(id)` sets activeId directly
- `enableFocus()` / `disableFocus()` toggle isFocusEnabled flag
- Export `FocusContext`, `FocusContextValue`

  <action>
Implement `parseKeypress(data: Buffer): { input: string; key: Key }` modeled on Ink's key parsing logic.

Handle these escape sequence families:
1. **CSI sequences** (`\x1b[`): arrows (A/B/C/D), page up/down (5~/6~), delete (3~), shift/ctrl/meta modifiers via parameter bytes (1;2 = shift, 1;5 = ctrl, 1;3 = meta)
2. **Simple control chars**: \r (return), \t (tab), \x1b alone (escape), \x7f (backspace), \x01-\x1a (ctrl+a through ctrl+z)
3. **Meta prefix**: \x1b followed by a printable char = meta+char
4. **Plain chars**: single printable characters with no modifiers

Return a default Key object with all booleans false, then set the matched ones to true.

Do NOT hand-roll from scratch — follow Ink's parseKeypress patterns for escape sequence table coverage. Cover the 15+ test cases in the behavior block.
