    - Returns { stdin, isRawModeSupported, setRawMode } from StdinContext

- Cleanup: remove listener, call setRawMode(false)
- Dependencies: [isActive] (wrap handler in useRef to avoid re-subscribing on handler change — same as Ink)

<task type="auto" tdd="true">
  <name>Task 2: Implement useFocus and useFocusManager hooks with tests</name>
  <files>src/hooks/use-focus.ts, src/hooks/use-focus-manager.ts, src/hooks/index.ts, src/hooks/__tests__/use-focus.test.ts, src/hooks/__tests__/use-focus-manager.test.ts</files>
  <behavior>
    useFocus:
    - Registers focus id on mount, unregisters on unmount
    - Returns { isFocused: boolean } where isFocused = (activeId === id)
    - With autoFocus: true, component gets focus immediately on mount
    - Uses a generated id if none provided

    useFocusManager:
    - focusNext() advances to next id in ordered list (wraps)
    - focusPrevious() goes to previous id (wraps)
    - focus(id) sets specific id as active
    - enableFocus() / disableFocus() toggle isFocusEnabled
  </behavior>
  <action>
**src/hooks/use-focus.ts:**
- `useFocus(options?: { autoFocus?: boolean; isActive?: boolean; id?: string }): { isFocused: boolean }`
- Generates a unique id via useRef + counter if not provided
- On mount: calls focusContext.add(id, { autoFocus })
- On unmount: calls focusContext.remove(id)
- Returns `{ isFocused: focusContext.activeId === id && focusContext.isFocusEnabled }`
- isActive option: if false, unregister from focus (same as Ink)

**src/hooks/use-focus-manager.ts:**
- `useFocusManager(): { enableFocus, disableFocus, focusNext, focusPrevious, focus }`
- Simple consumer of FocusContext — returns the navigation methods directly

**src/hooks/index.ts:**

- Re-export Key type from types.ts

**Tests:** Create a test wrapper providing FocusContext with a mock implementation. Verify registration, auto-focus, navigation order, and wrapping behavior.
