<task type="auto">
  <name>Task 2: Bridge integration tests against real Rust binary</name>
  <files>src/bridge/__tests__/ipc-child.test.ts</files>
  <action>
Create integration tests that spawn the actual `tui-engine-renderer` binary. These tests require `cargo build` to have been run first. Add a `beforeAll` that builds the binary: `execSync('cargo build', { cwd: 'crates/renderer' })` with a generous timeout (60s). If cargo is not available, skip all tests with `describe.skipIf`.

IMPORTANT: The Rust binary expects a real terminal (TTY) for crossterm. In CI/test environments without a TTY, the binary may fail to enter raw mode. Handle this:
- The tests should detect if no TTY is available and skip gracefully.
- Alternatively, the Rust binary could detect `!isatty(tty_fd)` and skip raw mode / alternate screen when no TTY is present (this is a reasonable addition to terminal.rs — if no TTY, just process stdin/stdout without terminal management). If the executor added this capability in Plan 01, use it. If not, mock tests should be used as fallback.

**Test cases:**

1. **Test: spawn and receive ready** (BRDG-01)
   - Create `IpcRendererBridge` with explicit binaryPath pointing to debug build.
   - Await the startup promise (or ready event).
   - Verify bridge state is 'ready'.
   - Call `shutdown()` to clean up.
   - Timeout: 5 seconds.

2. **Test: send render frame and receive ack** (BRDG-01, BRDG-04)
   - Spawn bridge, wait for ready.
   - Call `enqueueRender(simpleWidgetTree)` where simpleWidgetTree is a minimal BoxNode with one TextNode child (use the same shape as protocol test fixtures).
   - Wait for the rendered ack (listen for the internal ack update, or add a small delay + check `lastAckedFrameId`).
   - Verify ack frameId matches.
   - Shutdown.

3. **Test: EOF causes clean exit** (BRDG-02)
   - Spawn the binary directly (not through bridge) using `child_process.spawn` with `stdio: ['pipe', 'pipe', 'inherit']`.
   - Wait for `{"type":"ready"}` on stdout.
   - Close stdin immediately with `child.stdin.end()`.
   - Wait for child exit event.
   - Verify exit code is 0 (clean exit, not crash).
   - Timeout: 3 seconds.

4. **Test: shutdown message causes clean exit** (BRDG-03)
   - Spawn bridge, wait for ready.
   - Call `shutdown()`.
   - Await the shutdown promise.
   - Verify child process has exited.
   - Verify bridge state is 'stopped'.

5. **Test: frame coalescing drops intermediate frames** (BRDG-04)
   - Spawn bridge with `maxFps: 5` (200ms intervals for easier testing).
   - Wait for ready.
   - Rapidly call `enqueueRender()` 10 times with different frame content.
   - Wait 500ms (enough for ~2 ticks).
   - Verify that NOT all 10 frames were sent — only the latest few should have been rendered.
   - This can be verified by checking that `lastAckedFrameId` is close to 10 (the last frameId) but the number of acks received is much less than 10.
   - Shutdown.

6. **Test: crash surfaces error** (BRDG-03)
   - This is harder to test without a way to crash the renderer. One approach: send an intentionally malformed message and check that the bridge surfaces an error. Or spawn the binary and kill it with SIGKILL, verify bridge reports crash state.
   - Spawn bridge, wait for ready.
   - `child.kill('SIGKILL')` (access internal child process reference if exposed, or add a method for testing).
   - Wait briefly, verify bridge state is 'crashed' and onError was called.

Use `vitest` with `describe`, `it`, `expect`. Set test timeout to 15 seconds per test (child process spawning is slow). Use `afterEach` to ensure bridge is shut down even on test failure.
