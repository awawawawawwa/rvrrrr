# Plan 2: DOM Types, Utilities & Text Measurement Pipeline

## Objective
Create the DOM type definitions and the complete text measurement pipeline (ansi-tokenizer → sanitize-ansi → squash-text-nodes → measure-text, plus wrap-text and get-max-width). These are leaf-level modules that the DOM and reconciler modules will depend on.

## Context
- All files are adapted from Ink v6.8.0 source (`https://github.com/vadimdemedes/ink/tree/master/src`)
- `ansi-tokenizer.ts` is 340 lines of Ink-native code with zero npm dependencies — copy as-is
- `squash-text-nodes.ts` has a circular type-only import on `dom.ts` — resolve by importing types from `../dom/types.ts`
- Text measurement uses `widest-line` (which internally uses `string-width`) to handle Unicode, emoji, CJK, and ANSI
- The `OutputTransformer` type from `render-node-to-output.ts` is inlined into `types.ts` to break the dependency on Ink's render pipeline
- **No import from Plan 3:** To avoid circular dependency (Plan 3 re-exports Plan 2's layout files), Plan 2 must be self-contained. In `types.ts`, use a minimal inline type for `DOMElement.style` (e.g. `Record<string, unknown>` or inline the minimal style shape). In `wrap-text.ts`, use `type TextWrapType = 'wrap' | 'truncate' | 'truncate-start' | 'truncate-middle' | 'truncate-end'` instead of `Styles['textWrap']`

## Task 1: Create src/dom/types.ts — DOM Node Type Definitions

Fetch Ink's `dom.ts` from `https://github.com/vadimdemedes/ink/blob/master/src/dom.ts` to extract type definitions. Create `src/dom/types.ts` containing ONLY the type definitions (not the runtime functions — those go in Plan 4).

The file must define:

```typescript
import {type Node as YogaNode} from 'yoga-layout';

export type OutputTransformer = (s: string, index: number) => string;

/** Minimal style shape for DOMElement; full Styles type lives in layout/styles.ts */
export type DOMElementStyle = Record<string, unknown>;

export type NodeNames =
  | 'ink-root'
  | 'ink-box'
  | 'ink-text'
  | 'ink-virtual-text'
  | '#text';

export type DOMElement = {
  nodeName: string;
  attributes: Record<string, DOMNodeAttribute>;
  childNodes: DOMNode[];
  internal_transform?: OutputTransformer;
  parentNode?: DOMElement;
  yogaNode?: YogaNode;
  internal_static?: boolean;
  staticNode?: DOMElement;
  isStaticDirty?: boolean;
  onComputeLayout?: () => void;
  onRender?: () => void;
  onImmediateRender?: () => void;
  internal_layoutListeners?: Array<() => void>;
  style: DOMElementStyle;
};

export type TextNode = {
  nodeName: '#text';
  nodeValue: string;
  yogaNode?: YogaNode;
  parentNode?: DOMElement;
  style: Record<string, never>;
};

export type DOMNode = DOMElement | TextNode;
export type DOMNodeAttribute = boolean | string | number;
```

**Critical — no import from Plan 3:** The `DOMElementStyle` type above is a minimal inline type; the full `Styles` type lives in `layout/styles.ts` (Plan 3). This avoids circular dependency.

Refer to Ink's `dom.ts` for the exact field list on `DOMElement`. Keep `internal_transform`, `onComputeLayout`, `onRender`, `onImmediateRender`, `internal_layoutListeners`, `isStaticDirty`, `staticNode`, `internal_static` — these will be used in later phases. Remove `internal_accessibility` (not in v1 scope).
