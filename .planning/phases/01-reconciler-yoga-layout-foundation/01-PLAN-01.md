# Plan 1: Project Scaffolding & Configuration

## Objective
Initialize the tui-engine project with package.json, TypeScript configuration, build tooling, test framework, and directory structure. Install all Phase 1 dependencies with correct versions.

## Context
- Greenfield project — start from scratch with `npm init`
- Must use `"type": "module"` in package.json for yoga-layout top-level await compatibility
- **Version alignment:** CONTEXT.md mentions react-reconciler 0.32.0, but research shows Ink v6.8.0 uses 0.33.0 — we use 0.33.0 to match Ink's battle-tested version and avoid upgrading later
- yoga-layout 3.2.1 requires `"moduleResolution": "bundler"` in tsconfig
- scheduler@^0.27.0 is a required peer of reconciler 0.33.0
- `"jsx": "react-jsx"` enables React 19 JSX transform (no `import React` needed in components)
- vitest is the test runner (native ESM support, works with top-level await)

**Important version notes:**

- `@types/react-reconciler` — community-maintained, may lag behind; use `@ts-expect-error` where needed
- Do NOT include `@types/scheduler` — the scheduler package ships its own types

## Task 2: Create tsconfig.json

Create `tsconfig.json` at project root:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "outDir": "dist",
    "rootDir": "src",
    "jsx": "react-jsx",
    "types": ["node"]
  },
  "include": ["src"],
  "exclude": ["node_modules", "dist", "test"]
}
```

Key choices:
- `"moduleResolution": "bundler"` — required for `yoga-layout/load` exports resolution
- `"module": "ESNext"` — required for top-level await
- `"jsx": "react-jsx"` — React 19 automatic JSX transform
