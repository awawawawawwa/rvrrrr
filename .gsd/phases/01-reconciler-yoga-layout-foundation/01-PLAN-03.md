# Plan 3: Layout Styles Module

## Objective
Create the styles module that defines the `Styles` type (all Yoga layout props) and the `applyStyles()` function that maps React props to Yoga node setters. This covers all flexbox, dimension, spacing, and position properties.

## Context
- Adapted from Ink v6.8.0 `src/styles.ts` (~500 lines)
- Ink's styles.ts imports from `yoga-layout` (runtime), `cli-boxes` (types), `type-fest` (types), `ansi-styles` (types)
- yoga-layout 3.2.1 exports both TypeScript enums AND legacy `Yoga.CONSTANT_NAME` constants — Ink uses the legacy style throughout, keep that
- The `Styles` type is a union of all layout and visual props that React components accept
- `applyStyles()` is called by the reconciler's `createInstance` and `commitUpdate` to sync React props → Yoga node
- Percentage values for width/height/flex-basis are supported via `setWidthPercent()` etc.
- `textWrap` is a style prop (not a Yoga prop) — it controls text wrapping behavior and is stored on the node but not applied to Yoga

### 1c. applyStyles() function

The main export function that applies a `Styles` object to a `YogaNode`. It must handle:

**Flex property mapping:**
```typescript
// flexDirection
if (style.flexDirection === 'row') {
  yogaNode.setFlexDirection(Yoga.FLEX_DIRECTION_ROW);
} else if (style.flexDirection === 'column') {
  yogaNode.setFlexDirection(Yoga.FLEX_DIRECTION_COLUMN);
} else if (style.flexDirection === 'row-reverse') {
  yogaNode.setFlexDirection(Yoga.FLEX_DIRECTION_ROW_REVERSE);
} else if (style.flexDirection === 'column-reverse') {
  yogaNode.setFlexDirection(Yoga.FLEX_DIRECTION_COLUMN_REVERSE);
}
```

**Dimension with percentage support:**
```typescript
if (typeof style.width === 'number') {
  yogaNode.setWidth(style.width);
} else if (typeof style.width === 'string') {
  yogaNode.setWidthPercent(Number.parseFloat(style.width));
}
```

**Padding/margin per-side with shorthand resolution:**
```typescript
// Padding shorthand hierarchy: padding → paddingX/paddingY → paddingTop/etc.
if (style.padding !== undefined) {
  yogaNode.setPadding(Yoga.EDGE_ALL, style.padding);
}
if (style.paddingX !== undefined) {
  yogaNode.setPadding(Yoga.EDGE_LEFT, style.paddingX);
  yogaNode.setPadding(Yoga.EDGE_RIGHT, style.paddingX);
}
// ... same pattern for paddingY, individual sides
```

**Gap properties:**
```typescript
if (style.gap !== undefined) {
  yogaNode.setGap(Yoga.GUTTER_ALL, style.gap);
}
if (style.columnGap !== undefined) {
  yogaNode.setGap(Yoga.GUTTER_COLUMN, style.columnGap);
}
if (style.rowGap !== undefined) {
  yogaNode.setGap(Yoga.GUTTER_ROW, style.rowGap);
}
```

**Position type:**
```typescript
if (style.position === 'absolute') {
  yogaNode.setPositionType(Yoga.POSITION_TYPE_ABSOLUTE);
} else {
  yogaNode.setPositionType(Yoga.POSITION_TYPE_RELATIVE);
}
```

**Border widths (for layout, not rendering):**
Borders affect Yoga layout — a node with `borderStyle` set needs border width applied:
```typescript
// When borderStyle is set, apply 1-unit border to enabled sides
if (style.borderStyle) {
  const topEnabled = style.borderTop !== false;
  const bottomEnabled = style.borderBottom !== false;
  const leftEnabled = style.borderLeft !== false;
  const rightEnabled = style.borderRight !== false;

  yogaNode.setBorder(Yoga.EDGE_TOP, topEnabled ? 1 : 0);
  yogaNode.setBorder(Yoga.EDGE_BOTTOM, bottomEnabled ? 1 : 0);
  yogaNode.setBorder(Yoga.EDGE_LEFT, leftEnabled ? 1 : 0);
  yogaNode.setBorder(Yoga.EDGE_RIGHT, rightEnabled ? 1 : 0);
}
```

**Overflow:**
```typescript
if (style.overflow === 'hidden') {
  yogaNode.setOverflow(Yoga.OVERFLOW_HIDDEN);
} else {
  yogaNode.setOverflow(Yoga.OVERFLOW_VISIBLE);
}
```

The function signature should be:
```typescript
export default function applyStyles(yogaNode: YogaNode, style?: Styles): void;
```

Copy Ink's exact implementation — it handles all edge cases including `undefined` cleanup (resetting props to defaults when they become undefined during updates).
